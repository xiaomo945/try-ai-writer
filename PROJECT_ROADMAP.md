## 📋 工作流程规范

### 每次迭代的标准流程
1. **规划**：明确本次迭代的任务清单
2. **执行**：逐个完成任务
3. **测试**：测试所有功能
4. **构建**：`npm run build` 确保无错误
5. **提交**：`git add -A && git commit -m "feat: ..."`
6. **推送**：`git push origin main`
7. **更新**：更新本文件的进度
8. **汇报**：告诉用户完成情况，询问是否继续

### 需要人工处理的事项（延后）
这些任务需要用户提供资源或决策，先记录下来，等用户有时间时统一处理：

1. **数据库配置**

   - 需要：获取DATABASE_URL
   - 推荐：Supabase（免费）/ Railway / PlanetScale

2. **第三方服务配置**
   - 需要：配置Creem支付（已完成）
   - 需要：配置Google Analytics
   - 需要：配置SEO工具

3. **设计资源**
   - 需要：品牌logo优化

4. **商业决策**
